# Unidad I · Banco de Evaluacion y Rubrica

## Objetivo

Evaluar dominio conceptual sobre sistemas de bases de datos, arquitectura, actores y ventajas del enfoque BD.

## Formato sugerido (100 puntos)

| Componente | Puntos |
|------------|--------|
| Teoria corta (conceptos clave) | 30 |
| Analisis de caso | 30 |
| Preguntas de aplicacion | 40 |

## Banco de preguntas

### Parte A · Teoria (respuesta corta)

1. Define base de datos y minimundo con tus palabras.
2. Diferencia entre dato, informacion y metadato.
3. Que es un SGBD y por que no basta solo con archivos planos.
4. Explica las 4 caracteristicas del enfoque de BD.
5. Que significa independencia logica y fisica de datos.
6. Diferencia DDL, DML y DCL con un ejemplo de comando para cada uno.

### Parte B · Caso guiado

Caso: una clinica gestiona pacientes, medicos y citas en hojas de calculo separadas.

1. Identifica 4 problemas del enfoque por archivos.
2. Propone como un SGBD reduce redundancia e inconsistencia.
3. Define dos vistas externas distintas: recepcion y direccion medica.
4. Menciona 3 restricciones de acceso por rol.

### Parte C · Aplicacion

1. Dibuja en texto la arquitectura ANSI/SPARC para el caso clinica.
2. Relaciona cada actor con una responsabilidad concreta.
3. Propone 5 metadatos del catalogo para la tabla PACIENTE.
4. Explica un escenario de concurrencia y como lo maneja el SGBD.

## Criterios de correccion rapida

- Exactitud conceptual: 40%
- Claridad y capacidad de explicacion: 30%
- Aplicacion al caso: 30%

## Errores frecuentes

- Confundir SGBD con base de datos.
- Confundir esquema conceptual con esquema fisico.
- Dar ejemplos de comandos SQL en categoria incorrecta.
